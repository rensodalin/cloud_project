AWS Service Role in Architecture Rationale / Why it was chosen
🔀 Application Load Balancer (ALB) Distributes HTTP/HTTPS traffic across EC2 instances Layer-7 routing, health checks, SSL termination, sticky sessions
🖥 EC2 + Auto Scaling Group Web server compute layer, scales to meet demand Dynamic scaling by CPU & request count; auto-replace failed instances
🗄 RDS MySQL (Multi-AZ) Relational database backend Managed failover, automated backups, encrypted at rest
🪣 S3 Object storage for files, assets, and logs Infinitely scalable, durable, versioned, encrypted at rest
🌐 VPC + Subnets Network isolation; public subnets for ALB, private for EC2 & RDS Defence-in-depth; minimise attack surface by keeping compute private
📤 NAT Gateway Private subnet outbound internet access Allows software updates without exposing EC2 to inbound internet
🔐 IAM EC2 instance profile, RDS monitoring role, VPC Flow Log role Least-privilege: granular policies per service, no root keys
📊 CloudWatch Metrics, alarms, log groups, and dashboard Central observability; SNS integration for proactive alerting
🔔 SNS Email alarm notifications Decoupled alerting; can fan out to email, SMS, Lambda
💻 SSM Session Manager Secure shell access to EC2 (no open port 22) Eliminates key management and reduces attack surface
🔧 Terraform Infrastructure as Code Repeatable, version-controlled, documented infrastructure
🐙 GitHub Version control, collaboration, CI/CD readiness Full audit trail, branching, pull-request workflow
